## Суть проекта

`AUTOMATION-LOWCONTENT-BOOK` — конвейер для производства **low-content книг** под
Amazon KDP (линованные блокноты, журналы, планировщики и т.п.). Цель: кодом готовить
готовые к загрузке файлы (interior.pdf + cover.png + crib.md), оставив человеку только
ручную загрузку в KDP. Полная стратегия — в `PROJECT-OVERVIEW.md`, структура данных —
в `STRUCTURE.md`.

**Красная линия:** никакой автоматизации браузера против KDP (бан). См. `AGENTS.md`.

---

## Разделение ролей

- **Архитектор** (чат, дорогая модель) — стратегия, ТЗ, проверка, логи.
- **Кодер** (терминал, дешёвая модель, напр. DeepSeek/Qwen через Claude Code CLI) —
  пишет код по `tasks/TASK.md`, отчитывается в `tasks/REPORT.md`.

Обмен — только через файлы. Каноническая файловая система описана в `AGENTS.md`.

## Стек (предварительно)

- **Интерьер PDF:** Python + `reportlab` (генерация по геометрии KDP).
- **Обложка:** `Pillow` (позже).
- **LLM (названия/описания):** Claude API или Ollama; есть `reusable/python/llm_client.py`.
- **Данные разведки:** Publisher Rocket / Book Bolt, Amazon autocomplete, Google Trends.

## Текущий фокус

Физика книги собрана и принята:

`data/niches/2026-06_test/output/2026-06-05/book-1/` = реальные `interior.pdf` + `cover.png`
от наших движков. book-2/3 — заглушки.

**book-1 готов к ручной загрузке в KDP:** `interior.pdf` + `cover.pdf` + `crib.md`.

### Гибридная обложка — все 4 кирпича готовы
1. **Арт** (ComfyUI/RunPod, внешний) — чистый фон без текста.
2. **Раскладка** — vision-модель (или архитектор вручную, «шаг А») выдаёт JSON: где/как
   ставить текст. Тест-кейс: `data/niches/2026-06_test/sources/layout_test.json`.
3. **Текст кодом** — `engine/text_layout.py`: рисует текст по JSON (авто-кегль/перенос/
   контраст/scrim). Тест-арт: `sources/images/front_art_test.png`.
4. **Разворот+PDF** — `engine/cover_generator.py` (front_image) + `engine/cover_to_pdf.py`.

**Следующее — «сборка» в один проход:** art + layout JSON → text_layout → вставить
готовый фронт как `front_image` в cover_generator (полный разворот) → cover_to_pdf.
Затем: автоматизация шага А через vision-API; мост ComfyUI→папка ниши; данные ниши
в `niche.yaml`; сборщик `crib.md` из паспорта; пакетная генерация серии.

## «Зона безопасности» наложения текста — ПРИНЯТО (2026-06-08)

Тест на реальной обложке (`tasks/1.png` арт без текста, `tasks/2.png` эталон) показал:
слепые архетипы A–E наезжают на занятый угол (halftone-сфера), т.к. движок про арт не знает.
Решение принято и реализовано: зрение (архитектор, «шаг А») задаёт `safe_box` + потолок кегля
ОДИН раз, дальше детерминированный веер A–E работает ВНУТРИ очищенной зоны. Ноль коллизий.

- `text_layout.py` — `max_font_pct` в спецификации → `max_font_size` в `_autofit_font`.
- `layout_variants.py` — `generate_variants(..., safe_box_pct=[x,y,w,h])`; `_build_spec`
  пересчитывает боксы архетипов относительно зоны. `safe_box_pct=None` → геометрия как раньше.

Подход SAFE признан пользователем правильным. Приёмка — `tasks/done/2026-06-08_safe_box/`.
Демо-сравнение: `tasks/lv_test_safe/full|safe/_gallery.png` (тест-папки, в git НЕ идут).

**git-чекпоинт принят** (`tasks/done/2026-06-08_git-checkpoint/ACCEPTED.md`): движки на GitHub.

**ПРИНЯТО (2026-06-08):** цепочка `engine/build_cover.py` (оркестратор) + фронт-fill в
`cover_generator` — передний PNG → полный разворот (фронт cover-fit до bleed, DPI-защита,
задник плоский) → `cover.pdf`+`cover.png` в `data/niches/<ниша>/output/<дата>/book-N/`.
Приёмка: `tasks/done/2026-06-08_build-cover-chain/ACCEPTED.md`. Тест-ниша `2026-06_snarky-notebook`.
Передний арт пока низкого разрешения (заглушка) — мягкий; крепкость даст реальный арт ≥300 DPI.

**Урок:** shell-песочница может показывать устаревший снимок репозитория — при расхождении
доверять прямым файловым инструментам (live-файл), не выводам sandbox.

**ПОВОРОТ ПОДХОДА (2026-06-08):** пользователь решил рисовать **полный разворот сам** в
редакторе (пиксель-в-пиксель под Amazon). Скрипт упрощается до «валидатор размера + обёртка
в PDF»: `config.yaml` (размер разворота в in/cm + путь) → сверка с `размер×300` (±1 px) →
совпало: `cover.png`+`cover.pdf` в дерево ниши; не совпало: `raise` с требуемым размером.
cover-fit/DPI-турникет ОТМЕНЕНЫ для этого потока; `cover_generator` остаётся для будущего
авто-пайплайна, но в простом режиме не используется.

**ПРИНЯТО (2026-06-08):** простой режим `wrap_validated_cover` в `build_cover.py` + `config.yaml`
(in/cm). Сверяет размер разворота (±1 px) → совпало: `cover.png`+`cover.pdf` в дерево ниши;
не совпало: `raise` с требуемым размером. `build_cover_chain` (авто-сборка) сохранён, но из
CLI не вызывается. Приёмка: `tasks/done/2026-06-08_simple-validate-wrap/ACCEPTED.md`.
