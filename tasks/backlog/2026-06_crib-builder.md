## Суть

`engine/crib_builder.py`: читает `niche.yaml` (+ параметры книги: номер, дата) и
генерирует `crib.md` — шпаргалку для ручной загрузки в KDP — в папку книги
`data/niches/<ниша>/output/<дата>/book-N/`.

## Содержимое crib.md (контракт)

- Название книги (по `title_formula` ниши) и subtitle.
- Описание в **HTML** (из `description_template`) — KDP принимает разметку.
- 7 backend-ключей (из `keywords`, контроль ≤50 символов на слот).
- Категории, цена (из паспорта).
- Настройки печати (trim, bleed, бумага — из `interior_spec` / KDP_SETTINGS).
- Блок «что грузить»: `interior.pdf` + `cover.pdf` в KDP; `cover.png` НЕ грузить.
- AI-disclosure напоминание (`data/shared/text/ai-disclosure.txt`).

Эталон формата — ручной `crib.md` book-1 (`2026-06_test/output/2026-06-05/book-1/`).

## Зачем сейчас

Прямая экономия ручного времени на каждой загрузке (Фаза 3 = 3 книги/день);
все данные уже структурированы в `niche.yaml`; задача дешёвая, без новых зависимостей.

## Критерии приёмки (эскиз)

1. Запуск на `2026-06_test` book-1 даёт crib.md, совпадающий по составу с ручным эталоном.
2. Ключ >50 символов → понятная ошибка/предупреждение.
3. Идемпотентность: повторный запуск не портит существующие файлы книги.
4. TODO-значения паспорта переносятся как явные `TODO`-маркеры, не маскируются.
