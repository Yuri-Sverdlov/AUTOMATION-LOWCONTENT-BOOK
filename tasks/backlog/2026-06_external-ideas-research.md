# Чужие open-source идеи для нашего проекта (разведка GitHub)

**Создано:** 2026-06-10 (архитектор) · **Статус:** research, не активно
**Цель:** не изобретать велосипед — взять готовые паттерны под Фазу 1 (разведка) и
Фазу 4 (обратная связь), которых у нас ещё нет. Все кандидаты отфильтрованы по
**красной линии KDP** (никакой автоматизации браузера/скрейпинга Amazon).

---

## ✅ Безопасные и полезные (рекомендую присмотреться)

### 1. kdp-scout — keyword-разведка через Amazon autocomplete + скоринг
Майнит ключи из **Amazon autocomplete** (наш безопасный источник из PROJECT-OVERVIEW §5),
оценивает их композитным алгоритмом, прикидывает BSR/продажи. Бесплатно, локально, CLI.
- **Что взять:** алгоритм скоринга ключевых слов + сбор autocomplete → кормит `niche.yaml`.
- **Осторожно:** часть про BSR-конкурентов может тянуть данные с Amazon-страниц — этот
  кусок НЕ берём (красная линия), берём только autocomplete + скоринг.
- repo: https://github.com/rxpelle/kdp-scout

### 2. sales-rank-estimator — BSR → «книг в день»
Чистая **математическая модель**: по числу Amazon Sales Rank оценивает продажи в день.
Никакого скрейпинга — просто формула.
- **Что взять:** модель BSR→продажи для Фазы 4 («выстрелила ли книга») в `published.csv`.
- repo: https://github.com/m3mnoch/sales-rank-estimator

### 3. trendspyg (или pytrends) — Google Trends для сезонности
Современная замена архивированного `pytrends` (закрыт в 2025). Ловит тренд/сезонность —
ровно «поймать нишу до пика», как в PROJECT-OVERVIEW §4.
- **Что взять:** модуль сезонности ниши (Google Trends, НЕ Amazon — безопасно).
- repo: https://github.com/flack0x/trendspyg · (классика) https://github.com/GeneralMills/pytrends

### 4. kdp-puzzle-generator — новый тип интерьера (word search)
Генерит print-ready PDF кроссвордов/word-search для KDP за секунды, темы/сложность.
- **Что взять:** идею параметрического генератора **нового интерьера** (судоку/word-search)
  рядом с нашим lined — расширяет библиотеку интерьеров.
- repo: https://github.com/RealCharlesWallace/kdp-puzzle-generator

### 5. book-generator — сквозной AI-пайплайн + market-research утилиты
Тулкит «контент → EPUB/PDF/HTML для KDP» с утилитами рыночной разведки. Больше про
контентные книги, но структура пайплайна и market-research полезны как референс.
- **Что взять:** паттерны рыночной разведки + структуру «контент→печатный файл».
- repo: https://github.com/wesleyscholl/book-generator

### 6. (бонус) kdp-scout Cover Generator — арт обложки на DALL-E + Pillow
Автор kdp-scout отдельно открыл генератор обложек (DALL-E 3 + рендер на Pillow).
- **Что взять:** референс по связке «ИИ-арт + Pillow-рендер» — пересекается с нашим
  гибридным конвейером обложки и art-palette.

## ⛔ Популярное, но ПРОТИВ нашей красной линии (НЕ брать)

Самые «раскрученные» KDP-репозитории — как раз опасные: автоматизируют браузер против
KDP/Amazon (бан аккаунта). Знать в лицо, чтобы не соблазниться:
- **auto-kdp**, **SpiderKDP** — автозагрузка/клики в кабинете KDP (Selenium/Playwright).
- **amazon-bestseller**, **Basic-Amazon-Rank-Tracker**, **amazon-search-rank**,
  **amazon-bestsellers-scraper** — скрейпинг страниц Amazon (Selenium).
> Их данные мы получаем легально: autocomplete (бесплатно) + покупные BSR (Publisher
> Rocket / Book Bolt), а файлы грузим в KDP руками. См. PROJECT-OVERVIEW §2, §6.

## Моё мнение (приоритеты)

Три кандидата закрывают именно наши пробелы (у нас сильная «производственная» часть, но
пустые Фаза 1 и Фаза 4):
1. **kdp-scout (autocomplete+скоринг)** → автозаполнение ключей `niche.yaml`.
2. **sales-rank-estimator** → объективная метрика «выстрелила ли книга» (Фаза 4).
3. **trendspyg/Google Trends** → сезонность, вход в нишу до пика.

`kdp-puzzle-generator` — отдельный трек «новый тип интерьера», когда захотим расширяться
за пределы линованных блокнотов.
