## Зачем (контекст, чтобы ты понимал замысел)

Сейчас `layout_variants.py` генерирует 5 слепых архетипов A–E с боксами в процентах
**от всего арта**. Проблема: движок не знает, что на арте есть занятый угол (например,
halftone-сфера в правом-верхнем), и часть вариантов (A, C) наезжает на него. Плюс
авто-фит в `text_layout._autofit_font` подгоняет шрифт по высоте бокса и иногда раздувает
кегль до неуместного размера.

Идея (аналогия из живописи): сначала задаём **негативное пространство** — прямоугольную
«зону безопасности», где тексту жить можно, — и только потом веер из 5 стилей пробуется
**внутри** этой зоны. Зрение (архитектор) даёт две вещи: `safe_box` и потолок кегля.
Движок остаётся детерминированным и слепым, но работает уже в очищенной области.

**Ключевое требование — обратная совместимость:** при `safe_box_pct=None` поведение
должно быть БИТ-В-БИТ как сегодня (та же геометрия, те же bbox). Это критерий приёмки №1.

## Что сделать

### Шаг 1 — `engine/text_layout.py`: поддержать потолок кегля

`_autofit_font(...)` сейчас стартует с `size = min(_FONT_SIZE_MAX, box_h)`.
Добавь необязательный аргумент `max_font_size`:

```python
def _autofit_font(
    text, font_path, box_w, box_h, max_lines,
    leading=1.2, tracking=0, max_font_size=None,   # ← новый, в КОНЕЦ сигнатуры
):
    ...
    cap  = max_font_size or _FONT_SIZE_MAX
    size = min(_FONT_SIZE_MAX, box_h, cap)          # ← было min(_FONT_SIZE_MAX, box_h)
```

В `render_layout(...)`, в цикле по элементам, прочитай из спецификации новый
необязательный ключ `max_font_pct` (процент от ВЫСОТЫ арта `img_h`) и передай в авто-фит:

```python
mf_pct        = elem.get("max_font_pct")
max_font_size = round(mf_pct / 100 * img_h) if mf_pct else None
font, lines, font_size = _autofit_font(
    text, font_hint, bw, bh, max_lines, el_leading, el_tracking, max_font_size
)
```

Если `max_font_pct` нет в спецификации — `max_font_size=None`, и поведение прежнее.

### Шаг 2 — `engine/layout_variants.py`: «зона безопасности» + потолок в архетипах

**2a.** В каждый словарь `ARCHETYPES` добавь поле `"max_font_pct"`. Значения по умолчанию
(можешь оставить ровно эти): A=12, B=11, C=11, D=12, E=11.

**2b.** В `generate_variants(...)` добавь необязательный аргумент `safe_box_pct`:

```python

    art_path, text, out_dir,
    archetypes=None, base_color="#FFFFFF",

```

При `safe_box_pct=None` использовать `[0, 0, 100, 100]` (вся обложка → старое поведение).
Прокинь `safe_box_pct` в `_build_spec`.

**2c.** Перепиши `_build_spec(...)` так, чтобы `box_pct` архетипа трактовался как проценты
**ВНУТРИ зоны безопасности**, а в спецификацию записывался уже пересчитанный абсолютный
`box_pct` от всего арта. Также прокинь `max_font_pct`:

```python
def _build_spec(archetype, text, img_w, img_h, base_color, safe_box_pct):
    sx, sy, sw, sh = safe_box_pct           # проценты от арта
    ax, ay, aw, ah = archetype["box_pct"]   # теперь проценты ВНУТРИ зоны
    abs_box = [
        sx + ax / 100 * sw,
        sy + ay / 100 * sh,
        aw / 100 * sw,
        ah / 100 * sh,
    ]
    ...
    elements[role] = {
        ...                                  # как сейчас
        "box_pct":      abs_box,             # ← пересчитанный
        "max_font_pct": archetype.get("max_font_pct"),
    }
```

Математическая проверка обратной совместимости: при `safe=[0,0,100,100]` формула даёт
`abs = [0+ax/100*100, ...] = [ax, ay, aw, ah]` — ровно исходный `box_pct`. То есть без
`safe_box` геометрия не меняется. (Потолок кегля при этом всё же применится — это ОК и
ожидаемо; см. критерий №1: сравниваем геометрию боксов, а не финальный кегль.)
