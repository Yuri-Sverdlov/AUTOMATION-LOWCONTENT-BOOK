# ACCEPTED — простой режим: валидатор размера разворота + обёртка в PDF

**Дата приёмки:** 2026-06-08 · **Принял:** архитектор

## Вердикт: ПРИНЯТО

`wrap_validated_cover()` в `engine/build_cover.py` + `config.yaml` (in/cm) приняты.

## Проверено (живыми файловыми инструментами; bash-песочница была несвежей)

- **Код:** компилируется; `ValueError` при несовпадении кидается ДО `makedirs`/`copy` —
  значит на ошибке файлы не создаются (подтверждено живым ре-прогоном: book-ERRTEST не создан).
- **config.yaml:** валиден; формулы `in → px = round(v*dpi)`, `cm → px = round(v/2.54*dpi)`.
- **Ветка ПРОХОДА:** book-2/cover.png — верный разворот (живой просмотр: залитый перёд,
  корешок, плоский задник), cover.pdf геометрия 15.4967×9.5″.
- **Без ресемпла:** копия через `shutil.copy` (побитово). Размер/хэш book-1↔book-2 в bash
  расходились — это устаревший снимок book-1 (716K, r0) в песочнице; реальный book-1 = 3.1 MB.
- **Ветка ОШИБКИ:** на 2400×2957 — ValueError «нужен 4649×2850», файлы не созданы.
- **cm:** эквивалентные см дают те же 4649×2850 (±0).
- **Движки:** `cover_to_pdf/text_layout/layout_variants` не тронуты; `cover_generator.py`
  несёт только ранее принятый r1-дифф (новых правок в этой задаче нет).

## Состояние пайплайна

Простой режим рабочий: пользователь рисует разворот пиксель-в-пиксель → `config.yaml` →
`wrap_validated_cover` сверяет размер и кладёт `cover.png`+`cover.pdf` в дерево ниши.
`build_cover_chain` (авто-сборка) сохранён для будущего, но из CLI не вызывается.
