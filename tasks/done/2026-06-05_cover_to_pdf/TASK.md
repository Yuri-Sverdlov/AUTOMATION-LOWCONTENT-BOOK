## Цель

KDP для печатной (paperback) обложки на загрузке принимает **PDF**, а наш
`engine/cover_generator.py` выдаёт PNG (300 DPI). Нужна обёртка: взять готовый
PNG-разворот и завернуть его в **одностраничный PDF точного физического размера**,
чтобы KDP принял файл без ошибки «wrong dimensions».

Это чисто технический шаг (формат), без дизайна. Главная сложность — PDF-страница
должна иметь **ровно** физический размер разворота, а картинка — заполнять её
край-в-край без полей.

Эталон: `reference/series-1/` — там cover был PDF с MediaBox **1116 × 684 pt**
(= 15.5 × 9.5 дюйма). Наш `output/cover_test.png` (4649×2850 px @ 300 DPI) должен
дать практически тот же MediaBox.

---

## Ключевая математика

PNG имеет пиксельный размер `(w_px, h_px)` и DPI = 300. Физический размер в пунктах
(1 inch = 72 pt):
```
page_w_pt = w_px / DPI * 72        # 4649 / 300 * 72 = 1115.76 pt
page_h_pt = h_px / DPI * 72        # 2850 / 300 * 72 = 684.00 pt
```
Картинка кладётся в позицию (0, 0) и растягивается на весь размер страницы
(page_w_pt × page_h_pt) — без полей, без масштабных искажений (соотношение сторон
сохраняется автоматически, т.к. размер страницы выведен из самой картинки).

> Ожидаемый MediaBox для серии 1: **≈ 1115.76 × 684 pt** — это в пределах ±1 pt от
> эталонных 1116 × 684. Приёмка проходит при отклонении **≤ 1 pt**.

## Что написать

Файл: **`engine/cover_to_pdf.py`**

API:
```python
def cover_png_to_pdf(
    png_path: str,
    pdf_out: str,
    dpi: int = 300,
) -> dict:                      # вернуть {page_w_pt, page_h_pt, w_px, h_px, dpi}
    ...
```

Поведение:
1. Открыть PNG через Pillow, прочитать `(w_px, h_px)`. DPI берём параметром (300),
   не доверяем метаданным PNG (мы видели 299.9994 — артефакт формата).
2. Посчитать `page_w_pt`, `page_h_pt` по формулам выше (именованные константы, без
   магических чисел).
3. Создать **одностраничный** PDF с размером страницы ровно `(page_w_pt, page_h_pt)`
   и вписать картинку в (0,0)…(page_w_pt, page_h_pt) край-в-край.
   - Рекомендация: `reportlab.pdfgen.canvas.Canvas(pdf_out, pagesize=(w,h))` +
     `drawImage(png_path, 0, 0, width=w, height=h)`. Можно `img2pdf`, если предпочитаешь
     (тогда добавь в requirements). Pillow `save(...,'PDF')` допустим, но проверь MediaBox.
4. Не пережимать/не ресемплить картинку без необходимости (качество не терять).
5. `if __name__ == "__main__":` — если есть `output/cover_test.png`, обернуть его в
   `output/cover_test.pdf` и вывести посчитанный MediaBox. Если PNG нет — сначала
   вызвать генератор (`from cover_generator import build_cover`) на `output/cover_test.png`.

Зависимость: `reportlab` уже есть; если выберешь `img2pdf` — добавь в `requirements.txt`.
`output/` создавать программно, не коммитить.

## Критерии приёмки (числа впиши в REPORT.md)

1. **Одна страница** в PDF. ✅/❌
2. **MediaBox** сгенерированного PDF (через pypdf) = `page_w_pt × page_h_pt` по формуле;
   для `output/cover_test.png` — в пределах **±1 pt** от **1116 × 684**. ✅/❌
   (впиши фактический MediaBox)
3. **Картинка заполняет страницу край-в-край**: нет белых полей. Проверь — например,
   через pypdf достань XObject Image, сверь, что её отрисованная область = размер
   страницы; ИЛИ отрендерь PDF в PNG (pymupdf) и убедись, что фон обложки доходит до
   краёв. Опиши как проверял. ✅/❌
4. **Эффективное разрешение** ≈ 300 DPI: `w_px / (page_w_pt/72)` ≈ 300 (±1). ✅/❌
5. **PDF валиден**: открывается в pypdf без exception, `len(pages)==1`. ✅/❌
6. **Параметрично**: оберни также `output/cover_test_200p.png` (200 стр.) → PDF;
   его MediaBox шире по ширине (впиши оба MediaBox — для 110 и 200 стр.). ✅/❌
7. **Соотношение сторон** картинки сохранено (нет растяжения): page_w/page_h ≈ w_px/h_px. ✅/❌

> Точного совпадения байт с эталонным PDF НЕ требуется. Требуется корректный
> физический размер страницы и картинка без полей.
