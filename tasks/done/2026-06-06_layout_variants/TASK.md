## Контекст

`engine/text_layout.py` уже умеет рисовать текст на арте по **одному** JSON-спеку.
Теперь нужен слой выше: по одному арту и тексту сгенерировать **несколько заведомо
разных** раскладок (архетипов), отрендерить каждую и собрать **галерею-контактник**,
чтобы человек (или позже LLM-судья) выбрал лучшую.

Важно: варианты должны различаться по **осмысленным осям**, а не случайным дрожанием
на пару пикселей. Поэтому архетипы заданы детерминированно (ниже), без рандома.

Тест-кейс (как у text_layout):
- Арт: `data/niches/2026-06_test/sources/images/front_art_test.png` (733×903).
- Текст заголовка: `Things I Do When Nobody’s Watching (Totally SFW Edition)`.
- «Запретная зона»: верх-право занят полусферой — варианты с текстом наверху НЕ должны
  залезать в правый-верхний угол (держать текст левее или ниже там, где фон чистый).

## Архетипы (5 штук, детерминированные)

Каждый архетип = свой JSON-спек для `text_layout.render_layout`. Различаются по осям
позиция / выравнивание / масштаб / ритм / обработка. Координаты `box_pct` =
[x, y, w, h] в процентах от арта.

| id | название | box_pct (title) | align | масштаб | tracking/leading | scrim |
|----|----------|-----------------|-------|---------|------------------|-------|
| A | center-hero | [10, 34, 80, 42] | left | крупный (заполнить рамку) | обычный | false |
| B | bottom-band | [8, 60, 84, 34] | left | средний | плотный (leading×0.95) | true |
| C | centered-airy | [12, 30, 76, 45] | center | средний | воздушный (leading×1.25) | false |
| D | top-left-block | [8, 30, 58, 40] | left | крупный | обычный | false |
| E | lower-center | [14, 55, 72, 38] | center | крупный | обычный, трекинг +доп. | false |

Это **базовый набор**; вынеси его в структуру (список dict) в начале файла, чтобы легко
менять. Цвет текста по умолчанию `#FFFFFF` (фон тёмный), но используй авто-контраст из
`text_layout`, если цвет не задан.

> `text_layout.py` сейчас может не поддерживать управление tracking (межбуквенный) и
> leading (межстрочный). Если их нет — **добавь параметры** `tracking` и `leading`
> в `text_layout.render_layout` (по умолчанию tracking=0, leading=1.2) и реализуй; это
> часть задачи. Если уже поддерживает — используй.

## Что написать

Файл: **`engine/layout_variants.py`**

API:
```python
def generate_variants(
    art_path: str,
    text: str | dict,            # строка заголовка ИЛИ dict ролей {title, subtitle, author}
    out_dir: str,
    archetypes: list[dict] | None = None,  # None → встроенный набор A..E
    base_color: str = "#FFFFFF",
) -> dict:                       # вернуть: список вариантов с путями PNG и их спеками
    ...
```

Поведение:
1. Для каждого архетипа собрать полный JSON-спек (на базе архетипа + текста + размера арта).
2. Отрендерить каждый через `text_layout.render_layout` в `out_dir/variant_<id>.png`.
3. Сохранить рядом сам спек: `out_dir/variant_<id>.json` (для воспроизводимости/выбора).
4. Собрать **галерею-контактник** `out_dir/_gallery.png`: все варианты уменьшены и
   выложены в сетку (например 1×5 или 2×3), под каждым — подпись с `id` и названием
   архетипа. Так человек видит все варианты на одном экране.
5. Вернуть словарь: для каждого варианта — id, название, путь PNG, путь JSON, итоговый
   кегль/строки (из text_layout).
6. `if __name__ == "__main__":` — прогнать на тест-кейсе ниши, выход в
   `output/layout_variants/`. Вывести сводку.

Зависимость: `Pillow` (есть). Импортируй `text_layout` как модуль. `output/` не коммитить.
