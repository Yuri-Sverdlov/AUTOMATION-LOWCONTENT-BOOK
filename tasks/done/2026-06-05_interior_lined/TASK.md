## Точная спецификация

Единицы reportlab — **пункты (pt), 1 inch = 72 pt**. Исходник в пикселях @ 300 DPI,
коэффициент перевода `pt = px * 72/300 = px * 0.24`. Начало координат reportlab —
**нижний-левый угол**.

### Страница
- Размер: **540 × 666 pt** (7.5 × 9.25"), No Bleed (обрез = trim, без вылетов).
- Фон: белый. Контент чёрно-белый.
- Всего страниц: **110**.

### Структура книги (контракт)
- **Страница 1** — декоративная картинка (front matter). В этом задании НЕ
  реализуем картинку: сделай страницу 1 **пустой** (только белый фон) и оставь
  TODO-комментарий, что сюда позже встанет front-matter image. Параметр
  `front_matter_image: str | None = None` — если путь задан, вставить картинку
  по центру; по умолчанию None → пустая страница.
- **Страницы 2–110** — одинаковые линованные (wide-ruled), описание ниже.

### Линованная страница (wide-ruled)
- Линий на странице: **26**.
- Цвет линий: **#999999** (RGB 153,153,153).
- Толщина линии: **0.48 pt** (= 2 px @ 300 DPI).
- Горизонтальная протяжённость линии (X), абсолютные координаты страницы:
  - left X = **28.35 pt** (118.125 px)
  - right X = **521.10 pt** (2171.25 px)
  - (ширина линии = 492.75 pt; это ровно между внутренним и внешним полями KDP)
- Вертикальные позиции линий (Y, **от верха** страницы):
  - первая линия: **24.30 pt** от верха (101.25 px)
  - шаг между линиями: **24.696 pt** (102.9 px)
  - 26 линий: Y_top(n) = 24.30 + n*24.696, n = 0..25
  - последняя (n=25): 641.70 pt от верха
- Перевод в координаты reportlab (от низа): `y_bottom = 666 - y_top`.
  - первая линия: y_bottom = 641.70; последняя: y_bottom = 24.30.

### Поля KDP (справочно — уже учтены в координатах линий)
Внутреннее/корешковое 0.39375" (28.35 pt), внешнее/верх/низ 0.2625" (18.9 pt).
Линии уже вписаны в безопасную зону — отдельно поля настраивать не нужно.
