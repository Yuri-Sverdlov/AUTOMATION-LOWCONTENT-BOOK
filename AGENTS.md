# AGENTS.md — Устав кодирующего агента

> Этот файл — **правила работы** для агента-кодера (терминал / Claude Code CLI).
> Он меняется редко. Прочитай его целиком один раз в начале работы над проектом.
> Конкретное задание — в `tasks/TASK.md`. Отчёт о выполнении пиши в `tasks/REPORT.md`.

---

## Кто есть кто (разделение ролей)

Проект ведут два исполнителя через файлы:

- **Архитектор** (в чате, дорогая модель) — проектирует, пишет задания в `tasks/TASK.md`, проверяет отчёты, ведёт `PROJECT_LOG.md` и `CONTEXT.md`. Код сам почти не пишет.
- **Кодер** (ты, в терминале, дешёвая модель) — выполняешь `tasks/TASK.md`: пишешь код, запускаешь, тестируешь, и **обязательно** заполняешь `tasks/REPORT.md`.

Весь обмен идёт **только через файлы** в репозитории. Не придумывай задачи сам — делай то, что в `TASK.md`, и честно отчитывайся, включая то, что не получилось.

## Канонические файлы (единая система для всех проектов)

| Файл | Кто пишет | Назначение |
|------|-----------|------------|
| `AGENTS.md` | архитектор | устав кодера (этот файл), правила и запреты |
| `CLAUDE.md` | архитектор | тонкий редирект на этот файл для Claude Code |
| `CONTEXT.md` | архитектор | память проекта: суть, стек, текущий фокус |
| `tasks/TASK.md` | архитектор | **одно** активное задание архитектор → кодер |
| `tasks/REPORT.md` | кодер | **один** текущий отчёт кодер → архитектор |
| `tasks/done/` | архитектор | архив выполненных TASK/REPORT |
| `PROJECT_LOG.md` | архитектор | append-only журнал всех сессий |

**Правило против рассинхрона:** «правда о состоянии» живёт ровно в двух местах —
`REPORT.md` (что сейчас) и `PROJECT_LOG.md` (что было). Не заводи параллельных
файлов статуса (STATUS / NOTES / NEXT и т.п.).

## Цикл работы (каждый прогон)

1. Прочитай `CONTEXT.md` → затем `tasks/TASK.md`.
2. Выполни шаги задания по порядку.
3. Прогони проверки из раздела «Критерии приёмки» в `TASK.md`.
4. Заполни `tasks/REPORT.md` по шаблону: что сделано, результаты проверок,
   что не вышло, вопросы архитектору.
5. Остановись. Не коммить и не пушь в git, **если в `TASK.md` не сказано прямо**.

## Технические правила

- **Язык:** Python 3.10+. Зависимости — через `pip install -r requirements.txt`
  (или `pip install <pkg> --break-system-packages` в песочнице).
- **Песочница окружения:** длительные/блокирующие процессы (серверы) не запускай
  напрямую в основном потоке CLI — они вешают агента. Запускай в фоне или отдельным
  скриптом и проверяй через отдельную команду.
- **Не выдумывай числа.** Если задание ссылается на эталон (`reference/`), бери
  значения оттуда, а не из головы.
- **Идемпотентность.** Скрипт можно запускать повторно без порчи прошлых результатов.
- **Кодировка** всех текстовых файлов — UTF-8.

## Красная линия KDP (нерушимо)

Этот проект — конвейер low-content книг под Amazon KDP. **Запрещено** писать любой
код, автоматизирующий браузер против KDP (Playwright/Selenium/Puppeteer против
amazon.com/kdp.amazon.com), screen scraping KDP, недокументированные API Amazon —
это ведёт к бану аккаунта. Код доходит до подготовки файлов (PDF/PNG) и шпаргалки и
останавливается; загрузку в KDP делает человек вручную. Подробности — в
`PROJECT-OVERVIEW.md`, раздел 2.
