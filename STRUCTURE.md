# Структура папок проекта (не-кодовая часть)

> Этот документ описывает, как организуются **данные и результаты** проекта —
> ниши, книги, ресурсы и журнал. Код и стандартные директории проекта здесь не
> описываются. Скрипты должны строить и читать дерево именно в этом формате.

## Дерево

```
data/
├─ shared/                       # общие ресурсы на весь проект
│   ├─ images/                   #   логотипы, фоновые текстуры, общие иконки
│   ├─ text/                     #   общие тексты, дисклеймеры (напр. AI-disclosure)
│   └─ fonts/                    #   шрифты, используемые в обложках/интерьерах
│
├─ published.csv                 # ЖУРНАЛ всех выпусков (дубль-чек + трекинг BSR)
│
└─ niches/
    ├─ 2026-05_gratitude-journal/        # ниша = дата старта + слаг (не "ниша 1")
    │   ├─ niche.yaml                    # ПАСПОРТ ниши (см. ниже)
    │   ├─ sources/                      # ресурсы конкретно этой ниши
    │   │   └─ images/
    │   └─ output/
    │       ├─ 2026-05-01/               # ISO-дата (сортируется хронологически)
    │       │   ├─ book-1/
    │       │   │   ├─ interior.pdf
    │       │   │   ├─ cover.png
    │       │   │   └─ crib.md           # шпаргалка для ручной загрузки в KDP
    │       │   ├─ book-2/
    │       │   │   ├─ interior.pdf
    │       │   │   ├─ cover.png
    │       │   │   └─ crib.md
    │       │   └─ book-3/
    │       │       ├─ interior.pdf
    │       │       ├─ cover.png
    │       │       └─ crib.md
    │       └─ 2026-05-02/
    │           └─ ...
    │
    └─ 2026-06_sudoku-seniors/
        └─ ...
```

## Правила именования (важно)

**Даты — только ISO `ГГГГ-ММ-ДД`** (`2026-05-01`).
Формат `ДД.ММ.ГГ` сортируется по алфавиту неправильно и ломает хронологию
папок и связку с журналом. ISO сортируется естественно.

**Ниша — `ГГГГ-ММ_слаг`** (`2026-05_gratitude-journal`), а не `ниша 1`.
Из имени сразу видно, *когда* запущена и *что* это. Номера через месяц
ничего не скажут.

**Книга — отдельная папка `book-1 / book-2 / book-3`**, а не 9 файлов в куче.
В каждой папке лежат три файла одной книги вместе (interior + cover + crib).
При ручной загрузке в KDP открываешь одну папку и видишь ровно эту книгу —
нельзя случайно взять обложку от другой книги.

**Расширение шпаргалки — `.md`** (или `.txt`). Шпаргалка хранит размеченный
текст, в т.ч. описание в HTML, поэтому фиксированное расширение обязательно.

## niche.yaml — паспорт ниши

Единый контракт, из которого кормится весь конвейер. Сменить нишу = подменить
один файл, а не переписывать скрипты.

```yaml
niche: gratitude-journal
audience: women 30-55
format: 6x9            # размер книги в дюймах
interior_type: prompt-journal   # lined | dot-grid | sudoku | tracker | prompt-journal ...
palette: ["#F5E1E8", "#7A5C61", "#FFFFFF"]   # цвета из топа ниши
fonts: ["Playfair Display", "Lato"]
title_formula: "{audience} + {type} + {hook}"
keywords:                # 7 backend-слотов KDP, до 50 символов каждый
  - "gratitude journal for women"
  - "..."
description_template: "templates/gratitude.html"
price: 6.99
```

## published.csv — журнал выпусков

Одна строка на каждую выпущенную книгу. Нужен для двух вещей: не допустить
дубль (главный триггер бана KDP) и отслеживать BSR во времени.

| date | niche | book | title | keywords | asin | bsr_w1 | bsr_w2 | bsr_w3 |
|------|-------|------|-------|----------|------|--------|--------|--------|

`asin` и `bsr_*` заполняются вручную после публикации и при еженедельном
анализе. По динамике BSR видно, какие выпуски «выстрелили» — их паттерны
переносятся в паспорт следующей ниши.
